# Sentiment-Based Product Recommendation System — Flask App

A complete Flask web application that provides AI-powered product recommendations using Collaborative Filtering and NLP Sentiment Analysis.

## 🌟 Features

- **Personalized Recommendations**: Get product recommendations based on collaborative filtering and sentiment analysis
- **Sentiment Analysis**: Analyze the sentiment of product reviews (Positive/Negative)
- **RESTful API**: Complete API endpoints for integration with other systems
- **Modern Web Interface**: Beautiful, responsive UI for easy interaction
- **Real-time Analysis**: Instant predictions using trained ML models

## 📋 Prerequisites

Before running the Flask app, you need to:

1. **Have the data file**: Place `sample30.csv` in the project directory
2. **Run the Jupyter notebook**: Execute all cells in `sentiment_recommendation_notebook(1).ipynb` to generate the required pickle files

The notebook will create a `pickle/` directory with these files:
- `sentiment_model.pkl` - Trained sentiment classification model
- `tfidf_vectorizer.pkl` - TF-IDF vectorizer for text processing
- `user_based_cf.pkl` - User-based collaborative filtering model
- `master_reviews.pkl` - Processed review dataset

## 🚀 Installation

1. **Install Python dependencies**:
```bash
pip install -r requirements.txt
```

2. **Download NLTK data** (if not already downloaded):
```python
import nltk
nltk.download('punkt')
nltk.download('stopwords')
nltk.download('wordnet')
nltk.download('omw-1.4')
```

## 📊 Running the Notebook

Before starting the Flask app, generate the model files:

1. Place `sample30.csv` in the project directory
2. Open `sentiment_recommendation_notebook(1).ipynb` in Jupyter
3. Run all cells sequentially (Cell → Run All)
4. Wait for the notebook to complete (creates `pickle/` folder)

## 🎯 Running the Flask App

Once the pickle files are generated:

```bash
python app.py
```

The app will start on `http://localhost:5000`

You should see:

## 🌐 Using the Web Interface

1. **Open your browser** and navigate to `http://localhost:5000`

2. **Get Recommendations**:
   - Enter a username from the system
   - Set number of candidates (10-50)
   - Set number of final recommendations (1-10)
   - Click "Get Recommendations"

3. **Analyze Sentiment**:
   - Enter a product review text
   - Click "Analyze Sentiment"
   - View sentiment prediction and confidence score

## 🔌 API Endpoints

### 1. Get Product Recommendations
```http
POST /api/recommend
Content-Type: application/json

{
  "username": "john_doe",
  "num_candidates": 20,
  "num_recommendations": 5
}
```

**Response**:
```json
{
  "username": "john_doe",
  "recommendations": [
    {
      "product": "Product Name",
      "positive_count": 45,
      "total_reviews": 50,
      "positive_ratio": 0.9
    }
  ],
  "total_candidates": 20
}
```

### 2. Analyze Sentiment
```http
POST /api/sentiment
Content-Type: application/json

{
  "text": "This product is amazing! Works perfectly."
}
```

**Response**:
```json
{
  "original_text": "This product is amazing! Works perfectly.",
  "processed_text": "product amazing work perfectly",
  "sentiment": "Positive",
  "confidence": 0.9234,
  "probabilities": {
    "negative": 0.0766,
    "positive": 0.9234
  }
}
```

### 3. Get Users List
```http
GET /api/users
```

### 4. Get Products List
```http
GET /api/products
```

### 5. Health Check
```http
GET /health
```

## 🛠️ Troubleshooting

### "No such file or directory: 'sample30.csv'"
- Make sure `sample30.csv` is in the project root directory

### "Error loading artefacts"
- Run the Jupyter notebook first to generate pickle files
- Ensure all cells complete without errors

### "User not found in the system"
- Use `/api/users` endpoint to get valid usernames
- Or check the dataset for available users

### Port 5000 already in use
- Change the port in `app.py`: `app.run(debug=True, port=5001)`

## 📝 How It Works

1. **Collaborative Filtering**: Finds products that similar users liked
2. **Sentiment Analysis**: Analyzes review text to determine sentiment
3. **Filtering**: Ranks products by positive sentiment ratio
4. **Recommendations**: Returns top-N products with highest positive sentiment

## 🎓 Technical Stack

- **Backend**: Flask (Python web framework)
- **ML Models**: Logistic Regression, TF-IDF Vectorization
- **Recommender**: User-Based Collaborative Filtering
- **NLP**: NLTK (text preprocessing)
- **Frontend**: HTML, CSS, JavaScript (Vanilla)

## 📈 Performance

The system processes:
- ~30,000 reviews
- 200+ products
- 20,000+ users

Response times:
- Sentiment analysis: < 100ms
- Recommendations: < 2s (depending on candidate size)
